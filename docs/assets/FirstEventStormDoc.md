## 文件目的

本文件提供參與者在會議前需要具備的基本業務理解。

本次討論範圍是：

> 從「建立任務」到「任務完成」的基本流程。

# 1. 專案管理中的基本概念

## 1.1 Project：專案

Project 是一組工作目標的容器。

一個 Project 通常代表：

- 一個產品版本
- 一個開發案
- 一個客戶需求
- 一個內部改善計畫
- 一個研究或文件整理任務集合

在 RonFlow v0.1 中，可以先把 Project 理解為：

> 用來收納任務、追蹤進度、聚合工作成果的基本單位。

範例：

- RonFlow v0.1 開源版
- 建立部落格系統
- 重構權限模組
- 導入 CI/CD
- 撰寫專案管理工具的事件風暴文件

---

## 1.2 Task：任務

Task 是 Project 底下可被追蹤的工作單位。

一個 Task 通常代表一件需要被完成的具體工作，例如：

- 建立 Project 功能
- 建立 Task 功能
- 撰寫 README 初稿
- 設計 Domain Event 基礎類別
- 研究 EF Core 與 Dapper 的使用邊界
- 撰寫驗收測試

在 RonFlow v0.1 中，Task 可以先被理解為：

> 一個有明確標題、狀態與完成意義的工作項目。

Task 不一定一開始就非常完整。它可能先是一個粗略想法，後續再逐步補充描述、驗收條件、負責人、期限或優先度。

---

## 1.3 Backlog：待整理工作池

Backlog 是目前尚未完成、可能需要被處理的工作項目集合。

Backlog 不只是「待辦清單」，它也可以包含：

- 產品功能
- Bug
- 技術債
- 文件工作
- 架構決策
- 研究項目
- 重構項目
- 測試補強

Backlog 的核心價值是：

> 讓想法、需求與待辦工作先有地方被保存，再逐步排序、討論、取捨與澄清。

Backlog 不是承諾清單，不代表裡面的每一項都一定會被實作。

---
