# Acceptance Criteria

Acceptance Criteria 是驗收條件，指的是某一項需求、故事或工作項目在什麼情況下可以被接受的具體標準。它通常會描述功能應該呈現的結果、必須遵守的規則，或幾個關鍵情境應該如何成立。

好的驗收條件必須夠明確，讓產品、開發與測試對「完成」有一致理解。它不是泛泛而談的願望清單，而是能幫助團隊做出判斷的依據，例如哪些輸入要被接受、哪些狀態轉換是合法的、哪些邊界案例不能失敗。

在 [ATDD](./atdd.md) 或 [BDD](./bdd.md) 脈絡中，Acceptance Criteria 常會進一步轉成可執行的驗收測試或情境描述。它也和 [Definition of Done](./definition-of-done.md) 有關，但更聚焦於單一需求本身是否成立。
