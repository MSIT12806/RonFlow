# TDD

TDD 是 Test-Driven Development 的縮寫，中文常譯為「測試驅動開發」。它是一種以測試作為設計起點的開發方式，典型節奏是先寫失敗的測試，再寫最小實作讓測試通過，最後整理設計與重構程式碼。

這個循環通常被稱為 Red, Green, Refactor。Red 表示先寫出會失敗的測試，確認需求尚未被滿足；Green 表示只寫足夠讓測試通過的最小程式；Refactor 則是在測試保護下改善結構、命名與重複。

TDD 的價值不只是增加測試數量，而是迫使開發者從使用介面、依賴邊界與行為需求的角度思考設計。當測試很難寫時，往往也代表設計耦合過高或責任分配不清。

如果實踐得當，TDD 能讓重構更有信心，並且讓系統逐漸形成小而清晰、容易替換與驗證的設計。
