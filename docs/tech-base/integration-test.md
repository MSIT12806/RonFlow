# Integration Test

Integration Test 是整合測試，重點在驗證多個元件、模組或外部依賴接在一起時，系統是否仍然能正確運作。它不像單元測試那樣只關注單一函式或類別，而是更在意邊界接合後的真實行為。

整合測試常會碰到資料庫、訊息系統、檔案系統、Web API、框架設定或跨層呼叫等場景。這類測試的價值在於，很多問題不是出在某個模組本身，而是出在模組之間的契約、設定或資料流動方式。

它通常會和 [ATDD](./atdd.md) 或 [Outside-In](./outside-in.md) 開發互補使用。前者幫助團隊從需求角度確認交付結果，後者則能在往內實作時，透過整合測試檢查系統各層是否真的接得起來。
