# Use Case

Use Case 指的是某個角色為了達成特定目標，和系統進行的一段有意義互動。它關心的不是按鈕名稱或 API 細節，而是使用者想完成什麼事情，以及系統應該如何回應。

在需求分析裡，Use Case 常會連接 [Actor](./actor.md)、[Command](./command.md) 與驗收討論。它可以幫助團隊把抽象的功能需求整理成較清楚的互動情境，並進一步推進到 [Acceptance Criteria](./acceptance-criteria.md) 或測試案例。

因此，Use Case 比較像是站在使用者目標來描述系統行為，而不是從資料結構或元件分工出發。
