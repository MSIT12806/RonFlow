# Sprint

Sprint 是 [Scrum](./scrum.md) 中的一個時間盒，代表團隊在一段固定期間內，專注完成一組共同承諾的工作。常見長度可能是一週、兩週或一個月，但重點不在時間本身，而在於這段期間內有明確目標，並以可交付成果為導向。

在 Sprint 開始前，團隊通常會從 [Backlog](./backlog.md) 中挑選已經相對清楚、且值得優先處理的項目。Sprint 進行中，團隊則盡量聚焦在完成這批工作，而不是不斷插入新的臨時需求。

Sprint 的好處是讓規劃、檢視與調整形成穩定節奏。不過，它並不是所有團隊都必須套用的框架；如果工作性質更適合持續流動式管理，像 [Kanban](./kanban.md) 可能會更貼近實際需要。
