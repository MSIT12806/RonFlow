# Aggregate

Aggregate 是領域模型中的一致性邊界。它把一組彼此高度相關、需要一起維持規則的物件包在同一個決策單位裡，讓系統知道哪些狀態變化必須一起被保護。

在實作上，Aggregate 常會負責接收 [Command](./command.md)、檢查規則，然後決定是否產生 [Domain Event](./domain-event.md)。它關心的是業務一致性，而不是資料表是否剛好長得像某個物件集合。

因此，Aggregate 不只是把幾個 Entity 包在一起而已。更重要的是界定「哪些規則一定要在同一個邊界內一起成立」，這也是它在 DDD 設計中最有價值的地方。
