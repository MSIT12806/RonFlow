# Workflow

Workflow 指的是一項工作從開始到完成所經過的狀態、規則與流動方式。它的價值不在於把流程描述得多漂亮，而在於讓團隊看得出工作目前在哪裡、接下來該往哪裡走，以及卡住的地方究竟發生了什麼事。

在軟體開發裡，Workflow 常會搭配 [Backlog](./backlog.md)、[Kanban](./kanban.md) 或 [Sprint](./sprint.md) 一起使用。例如某個項目可能先待在 Backlog，準備清楚後進到 Ready，處理中變成 In Progress，最後經過 Review 才進到 Done。

好的 Workflow 不是製造更多儀式，而是降低狀態不透明帶來的成本。當工作流動方式足夠清楚，團隊就更容易對齊優先順序、辨識瓶頸，並針對流程做出具體調整。
