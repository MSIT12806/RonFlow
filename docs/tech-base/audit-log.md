# Audit Log

Audit Log 是稽核紀錄，重點在於留下可查驗、可追溯的變更證據。它通常會記錄誰在什麼時間做了什麼操作，必要時還會包含變更前後的值、來源、系統身分或其他稽核資訊。

相較於 [Activity Log](./activity-log.md) 偏向協作可讀性，Audit Log 更強調合規、追責與事後檢查。它常出現在權限管理、金流、資料異動、後台操作與企業系統等需要高可追溯性的場景裡。

Audit Log 也不一定等於 [Domain Event](./domain-event.md)。某些稽核紀錄只是為了證明某個變更曾經發生，不代表那個變更本身在業務語意上值得被視為核心事件。
