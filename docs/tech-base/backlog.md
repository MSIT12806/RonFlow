# Backlog

Backlog 指的是一份持續演進的工作清單，裡面收斂了所有可能讓產品更接近目標的需求、想法、修正、技術工作與研究項目。它不是單純把待辦事項堆在一起，而是讓團隊可以在同一個地方討論優先順序、範圍取捨與下一步投資方向。

在敏捷開發脈絡裡，Backlog 往往會隨著新資訊不斷被重新排序。某些項目會被拆小、補充驗收條件，某些項目則可能被延後、合併，甚至直接刪除。重點不在於清單是否穩定，而在於它是否始終對準目前最重要的目標。

如果專案有明確的 [Sprint](./sprint.md) 或 [Workflow](./workflow.md)，Backlog 通常會是最上游的來源。它幫助團隊知道有哪些工作值得被看見，哪些工作已經準備好往下一個狀態推進。
