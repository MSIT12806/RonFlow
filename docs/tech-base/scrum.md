# Scrum

Scrum 是一種敏捷開發框架，用來幫助團隊在高度不確定的環境中，以固定節奏逐步交付有價值的成果。它不是完整的軟體工程方法，而是一套用來管理工作透明度、檢視節點與調整節奏的框架。

Scrum 通常以 Sprint 為基本單位。每個 Sprint 都是一段固定時間，團隊在這段期間內聚焦完成一批明確目標，並在結束時檢視成果、回顧合作方式，調整下一輪計畫。

Scrum 常見的結構包含角色、事件與工件。角色如 Product Owner、Scrum Master、Developers；事件如 Sprint Planning、Daily Scrum、Sprint Review、Sprint Retrospective；工件則包含 Product Backlog、Sprint Backlog 與 Increment。

它的價值不在於開更多會，而在於讓需求排序、進度狀態、阻礙與交付結果保持透明，讓團隊能更早做出正確調整。
