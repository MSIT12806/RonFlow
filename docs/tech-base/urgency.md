# Urgency

Urgency 指的是一項工作是否具有立即處理的時間壓力。它關心的重點不是「這件事長期來看有多重要」，而是「如果現在不處理，會不會立刻造成明顯後果」。

在專案管理情境裡，Urgency 常見於 production bug、資安問題、客戶重大升級、法規期限或其他會打斷原有節奏的事件。這類工作不一定在長期規劃中排名最高，但可能因為時效性而必須優先插入當前流程。

因此，Urgency 通常應該和 [Priority](./priority.md) 區分。Priority 偏向相對重要性，Urgency 偏向時間壓力；兩者混在一起時，團隊很容易無法分辨一項工作究竟是重要、緊急，還是兩者兼具。
