# Domain Model

Domain Model 指的是系統中對業務概念、規則、狀態與關係的核心抽象。它不是單純把資料表欄位包成類別，而是要把問題領域真正重要的語意與限制表達出來，讓系統行為能貼近真實業務。

一個好的 Domain Model 會把關鍵概念命名清楚，並把重要規則留在模型裡，而不是散落在畫面、控制器或資料庫腳本之間。這樣做的好處是，當需求改變時，團隊更容易找到規則真正應該被調整的地方。

它和 [Domain Modeling](./domain-modeling.md) 有關，但兩者並不完全相同。Domain Modeling 比較偏向建模過程與方法；Domain Model 則是建模之後，實際落在系統裡的那套概念與規則表達。
