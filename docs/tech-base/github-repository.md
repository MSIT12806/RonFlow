# GitHub Repository

GitHub Repository 指的是 GitHub 平台上的一個程式碼儲存庫。它通常是專案的主要協作單位，裡面不只放原始碼，也會包含文件、版本歷史、Issue、Pull Request、Actions 工作流與其他協作資訊。

從版本控制角度來看，repository 保存了每一次提交的歷史，因此團隊可以追蹤變更、比較差異、回溯問題來源，並透過 branch 與 pull request 來管理併入流程。

從團隊協作角度來看，repository 也是討論與知識沉澱的入口。需求討論可能出現在 Issue，設計決策可能記錄在 PR 對話，文件與操作說明則會留在 README 或 docs 目錄中。

因此，當文章提到「GitHub repo」時，通常不只是指一個放程式碼的地方，而是整個專案協作與演進紀錄的載體。
