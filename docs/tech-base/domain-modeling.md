# Domain Modeling

Domain Modeling 是領域建模，指的是把真實業務世界中的概念、規則、關係與語意整理成一個可以共同理解的模型。這個模型不只是畫圖或命名而已，而是要幫助團隊建立一致的語言，並讓系統設計貼近實際問題。

在領域建模過程中，常會去辨識關鍵名詞、角色、事件、狀態轉換與商業規則，並思考哪些概念應該成為實體、值物件、流程節點或聚合邊界。這能避免系統只是照著資料表堆功能，卻無法反映真正的業務脈絡。

好的領域模型能降低溝通成本。當需求討論、測試情境、程式碼命名與文件都使用相同語彙時，誤解會顯著減少，重構與擴充也更容易。

它常出現在 Domain-Driven Design、事件風暴、需求探索與 BDD 等脈絡中，因為這些方法都很在意「系統到底在解決哪個真實問題」。
