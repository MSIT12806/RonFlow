# Domain Event

Domain Event 指的是在業務領域中，已經發生且具有明確語意的重要事實。它描述的不是「系統做了什麼技術操作」，而是「業務世界裡發生了什麼值得被記住的事情」，例如任務被建立、驗收被通過、訂單被取消。

判斷一個事件是否是 Domain Event，關鍵在於它是否對業務流程、規則、狀態或後續行為有實質意義。如果只是某個欄位被改掉、某個畫面被點擊、某個排序被調整，那未必就是 Domain Event；它也可能只是資料異動、畫面操作，或記錄在 [Activity Log](./activity-log.md) 裡比較合適。

在 [Event Storming](./event-storming.md) 中，辨識 Domain Event 是核心工作之一。因為一旦事件抓得不準，後面的命令、規則、模型邊界與系統設計都很容易跟著偏掉。
