# BDD

BDD 是 Behavior-Driven Development 的縮寫，中文常譯為「行為驅動開發」。它關注的不是系統內部怎麼寫，而是系統對外應該表現出什麼行為，尤其是從使用者、業務角色與情境的角度來描述需求。

BDD 常用 Given、When、Then 這種結構來表達情境。這樣的寫法可以讓產品、測試與開發用更接近自然語言的方式對齊需求，減少規格落差。它的重點不是格式本身，而是透過具體情境把模糊需求變成可驗證的共同理解。

BDD 往往會把需求討論、驗收條件與自動化測試串在一起。當情境描述足夠清楚時，團隊就更容易把它轉成驗收測試、應用層流程與底層設計。

它和 TDD 有關，但關注層次不同。TDD 比較偏向開發者如何用測試驅動設計；BDD 比較偏向團隊如何用行為與語言驅動需求對齊。
