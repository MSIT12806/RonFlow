# ATDD

ATDD 是 Acceptance Test-Driven Development 的縮寫，中文常譯為「驗收測試驅動開發」。它強調在實作之前，先定義系統要交付什麼結果，並把這些結果具體化成驗收條件或驗收測試。

ATDD 的重點是從使用者價值與業務成果出發，而不是先從技術細節出發。團隊通常會先討論「什麼情況算完成」、「哪些情境一定要成立」、「哪些規則不能被破壞」，再把這些內容轉成可驗證的測試或範例。

這種方式有助於避免做出功能存在、但其實不符合使用者期待的系統。它也能讓產品、測試與開發在實作前就先對齊完成定義。

ATDD 常與 BDD 互補。BDD 偏重行為描述與語意對齊，ATDD 偏重驗收結果與交付標準；兩者都能幫助需求更早變得清晰。
