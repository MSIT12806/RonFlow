# XP

XP 是 Extreme Programming 的縮寫，中文常翻成「極限程式設計」。它是一套強調工程實踐的敏捷方法，核心目標是在需求持續變動的情況下，仍然維持高品質與可持續交付。

XP 重視快速回饋。團隊會透過短週期交付、小步提交、持續整合與頻繁測試，盡早發現理解錯誤、設計問題與整合風險，而不是等到專案後期才一次爆發。

XP 也重視程式碼品質。常見實踐包括測試先行、重構、結對程式設計、簡單設計、共同程式碼擁有權，以及以可讀性與可維護性為優先的命名與結構調整。

如果把 Scrum 比喻為管理節奏與工作流的框架，XP 更像是一套面向日常開發的工程操作法。兩者經常一起使用：Scrum 管流程，XP 管技術實踐。
