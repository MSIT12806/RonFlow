# Definition of Done

Definition of Done 常縮寫為 DoD，指的是團隊對「什麼狀態才算真正完成」的共同標準。它不是一句模糊的完成感，而是一組可以被檢查的條件，例如程式已合併、測試已通過、文件已更新、驗收條件已滿足等。

有了明確的 Definition of Done，團隊就不容易出現「我以為這樣就算好了」的落差。它能降低交接摩擦，也能讓 [Review](./workflow.md) 與驗收更一致，避免工作被過早標記為 Done，卻在之後補出大量尾巴。

它和 [Acceptance Criteria](./acceptance-criteria.md) 有關，但兩者不完全一樣。Acceptance Criteria 比較偏向某一項需求應滿足的具體條件；Definition of Done 則是整個團隊對交付完成品質的共通底線。
